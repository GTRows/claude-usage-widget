# Claude Usage Widget 1.11.0-gtrows.1

GTRows fork release. The previous 1.10.0-gtrows.1 introduced the
`claude-usage` companion CLI; this build focuses on the tray mascot
experience and a handful of CLI conveniences that landed after the
last tag.

## Highlights

### Tray mascot, reworked

- **Crisper icon.** The mascot now renders on a 64 px canvas so the
  Windows tray (and its HiDPI variants) have enough detail to downscale
  without turning the face into a smudge.
- **Visible motion.** Every animation now uses larger deltas —
  breathing pulse, head tilt while idle, full-width eye darts, and a
  red/amber color flash with scale pulse near the limit.
- **Animate mascot now means mascot.** Turning the setting on dedicates
  the tray to the mascot cycle instead of appending it after the big
  number.
- **Two new settings.**
  - *Mascot animation length* (1–60 s, default 2 s) scales the full
    play to the requested duration.
  - *Pause between plays* (0–600 s, default 10 s) holds the big-number
    tray icon between animations so the tray is not in constant motion.
- **Calm by default.** First render is the number; the mascot only
  plays after the first pause elapses.

### CLI

- `claude-usage history --since N --format csv|json [--output FILE]`
  reads the desktop widget's stored history across platforms without
  going through Electron.
- `claude-usage doctor` checks credentials (env vs file), widget store
  presence, and live API reachability in one command.
- `claude-usage prompt --segments 5h,7d,opus,sonnet,extra` lets shell
  prompts pick which usage segments to render.
- `claude-usage prompt --cache N` caches the latest response for N
  seconds so shell prompts redraw without hitting the API on every
  keystroke. Docs under `docs/prompts/` lead with `--cache 60`.

### Settings / Data

- *Export history* now offers a date-range selector (All / 24h / 7d /
  30d / 90d) before writing CSV or JSON.

### Tests

- 72 passing vitest cases (up from 55 in the last release), covering
  the CLI api/config/render/widget-store modules, the shared thresholds
  / history / formatters / version / settings-schema modules, and the
  new settings clamps.

## Known issues

- App icons and installer art are still inherited from upstream. A
  GTRows-specific icon pass is on the roadmap.
- macOS builds are unverified — no GTRows artifacts are attached for
  that platform yet.
